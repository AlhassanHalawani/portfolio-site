WireGuard + Dynamic DNS: practical tips

2025-07-20 · WireGuard, Cloudflare, DDNS

A few notes on keeping your VPN reachable even as your public IP rotates…

Here are the small things that make a big difference when you rely on WireGuard at home while your WAN IP changes.

1) Use a DDNS that you control

I use a tiny script with the Cloudflare API that updates an A record whenever my public IP changes. Any equivalent will do—just make it cron‑friendly and idempotent.

2) Endpoint = hostname, not IP

[Peer]
Endpoint = vpn.example.com:51820
PersistentKeepalive = 25

3) MTU, MTU, MTU

Start with 1420, test, then go lower if you see fragmentation. Mobile networks often like 1280.

4) Health checks

Export wg show to Prometheus or push a tiny heartbeat to a monitoring endpoint so you actually notice when the tunnel dies.

5) Bonus: k3s + WG

Expose the Kubernetes API only over WireGuard. Your future self will sleep better.
