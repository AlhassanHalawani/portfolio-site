Getting started with k3s on Raspberry Pi 5

2025-08-01 · k3s, Raspberry Pi, Nginx

These are quick notes for installing k3s on a Pi 5 cluster and exposing services via Nginx Ingress with cert-manager…

This is my no-fluff way to get a Pi 5 cluster running k3s with a proper Ingress, TLS from Cloudflare, and a sane GitOps flow. It’s opinionated and repeatable.

1) Base OS and prerequisites

  • Flash Raspberry Pi OS 64‑bit (Lite) and enable SSH. Set a static DHCP lease on your router.
  • Update firmware: sudo rpi-eeprom-update -a then sudo reboot.
  • Harden SSH: disable password auth, add your key, and change the default user.

2) Install k3s (server)

curl -sfL https://get.k3s.io | sh -s - --write-kubeconfig-mode 644 --disable traefik

Grab the node token for future agents: /var/lib/rancher/k3s/server/node-token.

3) Join agents

K3S_URL=https://<server-ip>:6443 K3S_TOKEN=<token> curl -sfL https://get.k3s.io | sh -

4) Nginx Ingress + cert-manager

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.15.3/cert-manager.yaml

Create a ClusterIssuer for Cloudflare DNS01 and you’re golden.

5) GitOps

Use a repo with apps/ manifests and a CI job that runs kubectl apply -k against your cluster through a WireGuard tunnel.
